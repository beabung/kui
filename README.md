2. 운영 브랜치 가이드

• `master` 브랜치:

  • 이 브랜치에 변경 사항을 push하기만 하면, GitHub Actions가 자동으로 빌드 및 배포를 수행합니다.

• `local` 브랜치:

  • 로컬 환경에서 작업하거나 테스트할 때 사용합니다.

  • 프로젝트 루트 경로에 있는 `local.sh` 파일을 실행하여 로컬 서버를 구동할 수 있습니다.

  • 실행 방법: `./local.sh`
